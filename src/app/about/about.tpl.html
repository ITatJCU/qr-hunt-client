Frequently Asked Questions

Why don't I need an account?

The web client uses a method known as browser fingerprinting to generate an identifier for the web browser without gathering any personal information. This methodology allows the system to identify, with up to 94% accuracy, which pages a player visits.

This technique is often used by mobile advertisers to generate targeted advertisements.

How is the game ranked?

Each player has a collected number of scans linked to their player id, and a time stamp for each. The server tasked with tracking all players compares the total number of codes collected and a duration of time between the first and last code scanned.

The players with a higher total of scans, in the shortest period of time win.

Can I scan the same code twice?

Yes, and it is encouraged! Scanning the same code changes the duration between first and last unique scans.

Find all the codes and race between them to reduce your time!

Is this a student project?

A student who was motivated by the idea volunteered themselves to implement QRHunt as pet project.

Developer Related Questions

What languages and frameworks are used?

The Client

The client makes use of a relatively new web orientated framework called AngularJS by leveraging the popular ngBoilerplate build management system in combination with the mobile-first front-end framework, Twitter Bootstrap.

The Server

The server is written in JavaScript for the synchronous event driven framework Node.js and currently running on a RaspberryPi somewhere in Townsville.

Development time was reduced by utilising the restify and Sequelize modules for Node.js.

Is the source code available?

All source code for the server and web orientated client are available for your viewing pleasure on GitHub at: QR Hunt Client and QR Hunt Server respectively.
